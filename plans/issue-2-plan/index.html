    Issue #2: Server scaffolding & WebSocket setup

    Summary

    Replace the stubbed plain-WebSocket server with a production-ready Express + Socket.io infrastructure. Set up room management, health checks, environment configuration, and containerization. This is the foundational layer that all real-time game features depend on.

    Root Cause Analysis

    The project currently has a minimal stubbed server under server/: - server/package.json — Uses plain ws (WebSocket) library, no Express, no Socket.io - server/src/index.js — Stubs for createServer(), startServer(), stopServer() - server/src/websocket.js — Stubs for WebSocket server creation and broadcasting - server/src/connection.js — Stubs for connection lifecycle management - No room management, no health checks, no CORS, no TypeScript, no Docker

    The issue explicitly requires Socket.io (rooms, namespaces, broadcasting) and Express (HTTP routes, middleware). The current ws-based stubs cannot fulfill these requirements without a full rewrite.

    Proposed Solution

    1. Replace the server directory with a TypeScript-based Express + Socket.io setup
    2. Keep backward compatibility with existing lobby/client types (src/types/lobby.ts)
    3. Add room management with in-memory state (sufficient for MVP; can migrate to Redis later)
    4. Add health endpoint and environment-based configuration
    5. Containerize with a multi-stage Dockerfile
    6. Update root package.json scripts for unified dev workflow

    Phase 2: Core Server (src/server.ts)

    1. Implement server factory: 
      import express from 'express';
import { createServer } from 'http';
import { Server as SocketIOServer } from 'socket.io';
import { loadConfig } from './config.js';
import { createHealthRouter } from './middleware/health.js';
import { createCorsMiddleware } from './middleware/cors.js';
import { registerSocketHandlers } from './socket/handlers.js';

export function createApp() {
  const app = express();
  const httpServer = createServer(app);
  const io = new SocketIOServer(httpServer, {
    cors: { origin: loadConfig().corsOrigin, methods: ['GET', 'POST'] }
  });

  app.use(createCorsMiddleware());
  app.use('/health', createHealthRouter());

  registerSocketHandlers(io);

  return { app, httpServer, io };
}

export async function startServer(port?: number) {
  const config = loadConfig();
  const { httpServer, io } = createApp();
  const actualPort = port ?? config.port;

  return new Promise<{ httpServer; io; port: number }>((resolve) => {
    httpServer.listen(actualPort, () => {
      console.log(`🚀 Server listening on port ${actualPort}`);
      resolve({ httpServer, io, port: actualPort });
    });
  });
}

export async function stopServer(httpServer, io?) {
  io?.close();
  await new Promise<void>((resolve) => httpServer.close(() => resolve()));
}

    Phase 4: Socket Handlers (src/socket/handlers.ts)

    1. Register Socket.io events: 
      export function registerSocketHandlers(io: SocketIOServer) {
  io.on('connection', (socket) => {
    socket.join('lobby');
    socket.emit('room:list', listRooms());

    socket.on('room:create', (data, callback) => {
      const room = createRoom(data?.name);
      socket.join(room.id);
      socket.to('lobby').emit('room:created', room);
      callback?.({ success: true, room });
    });

    socket.on('room:join', (roomId, callback) => {
      const room = joinRoom(roomId, { id: socket.id });
      socket.join(roomId);
      socket.to(roomId).emit('room:playerJoined', { playerId: socket.id });
      callback?.({ success: true, room });
    });

    socket.on('room:leave', (roomId) => {
      leaveRoom(roomId, socket.id);
      socket.leave(roomId);
      socket.to(roomId).emit('room:playerLeft', { playerId: socket.id });
    });

    socket.on('disconnect', () => {
      // Clean up rooms where this socket was a player
    });
  });
}

      export function createCorsMiddleware() {
  return (req, res, next) => {
    res.header('Access-Control-Allow-Origin', loadConfig().corsOrigin);
    res.header('Access-Control-Allow-Methods', 'GET, POST');
    res.header('Access-Control-Allow-Headers', 'Content-Type');
    next();
  };
}

    Test Strategy

    Unit Tests (new — server/src/**/*.test.ts)

    • server.test.ts: Server factory creates Express app + Socket.io; start/stop lifecycle; port binding
    • rooms.test.ts: createRoom generates unique IDs; joinRoom adds player; leaveRoom removes player; listRooms filters active; empty room auto-deletes
    • handlers.test.ts: Socket.io client connects; receives room list; can create room; can join room; receives broadcast on player join; disconnect cleans up
    • config.test.ts: loadConfig returns defaults when env vars missing; parses PORT as number; loads from process.env
    • health.test.ts: GET /health returns { status: 'ok', uptime: number }

    Integration Tests

    • Full client → server → Socket.io round-trip using socket.io-client
    • Room creation flow: client connects → creates room → second client joins → both receive updates
    • Disconnect cleanup: client in room disconnects → room removed if empty

    Edge Cases

    • Port already in use → should throw with clear error
    • Invalid CORS origin → should still allow configured origin
    • Rapid create/join/leave → room state remains consistent
    • Socket reconnect → rejoins lobby, receives fresh room list

    Risks & Mitigations

    Risk Mitigation
    Plain ws → Socket.io migration breaks existing tests The existing stubs have no tests; this is a greenfield rewrite of the server directory
    Socket.io version compatibility with client Pin both socket.io and socket.io-client to same major version (^4.7)
    In-memory rooms lost on server restart Acceptable for MVP; document Redis migration path in code comments
    CORS blocking client in dev Default CORS_ORIGIN=http://localhost:5173 matches Vite default; document override
    TypeScript ESM + NodeNext resolution quirks Use .js extensions in imports (NodeNext requirement); test with tsx in dev and compiled output in CI
    Docker build slow due to npm ci Use --mount=type=cache or lock package-lock.json to minimize rebuilds
    Port conflict with other services Default to 3001 (Vite uses 5173); make PORT fully configurable via env
