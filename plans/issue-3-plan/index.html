    Issue #3: Chess game engine — move validation, game state, FEN/PGN

    Summary

    Build a complete chess engine that handles board representation, move validation for all pieces, special moves (castling, en passant, promotion), check/checkmate/stalemate detection, turn management, FEN import/export, and PGN export. The engine must be usable both server-side (authoritative validation) and client-side (UX previews).

    Root Cause Analysis

    The project currently has only stubbed engine code: - src/engine/types.ts — TypeScript type definitions exist but use shorthand naming ('p', 'w') while test files expect longhand ('pawn', 'white') - src/engine/board.ts — Basic board utilities (coordinate conversions, board creation) are implemented - src/components/ChessBoard.tsx — Deliberately renders a 7×7 grid with no pieces (RED-phase stub) - No move generation, no FEN parser, no game state manager, no PGN support - chess.js v1.4.0 is already listed as a dependency but not yet integrated

    Proposed Solution

    Reuse chess.js as the core engine and wrap it in a project-specific API layer. This avoids re-implementing complex chess rules (pin detection, insufficient material, threefold repetition, 50-move rule, etc.) and gives us a well-tested foundation. Build the following modules:

    1. src/engine/chess.js — Main ChessEngine class wrapping chess.js Chess
    2. src/engine/fen.ts — FEN parsing/generation utilities (wraps chess.js + custom format conversions)
    3. src/engine/moves.ts — Move generation and validation (wraps chess.js + exposes per-square legal moves)
    4. src/engine/game.ts — Game state management: createGame(), makeMove(), undoMove(), getGameResult()
    5. src/engine/special.ts — Special moves: castling, en passant, pawn promotion helpers
    6. src/engine/pgn.ts — PGN generation (generatePGN()) and SAN notation (moveToSAN())
    7. src/components/ChessBoard.tsx — Fix the 7×7 stub → proper 8×8 interactive board with drag-and-drop, legal-move highlights, check indicators, and promotion picker

    Key Decision: Naming Convention

    chess.js uses shorthand ({ type: 'p', color: 'w' }). The existing types.ts already defines this convention. However, several test files (board.test.ts, moves.test.ts, fen.test.ts, game.test.ts, special.test.ts, pgn.test.ts) use longhand ({ type: 'pawn', color: 'white' }).

    Decision: Use chess.js shorthand as the canonical internal format. Update the longhand test expectations to match, or provide a lightweight toLonghand() / toShorthand() conversion helper if needed for external APIs.

    Implementation Steps

    Phase 1: Core Engine Wrapper

    1. Create src/engine/chess.js extending/wrapping chess.js Chess class
    2. Constructor: new ChessEngine(fen?) — loads starting position or given FEN
    3. .move(input) — accepts SAN string ('e4'), object ({ from, to, promotion }), or long algebraic; returns true/false
    4. .fen() — exports current position as FEN string
    5. .turn() — returns 'w' or 'b'
    6. .get(square) — returns { type, color } or null
    7. .moves(options?) — returns array of SAN strings or verbose Move objects
    8. .isLegal(move) — validates a move without executing it
    9. .inCheck(), .isCheckmate(), .isStalemate(), .isDraw() — game-state queries
    10. .undo() — reverts last move, returns undone move or null
    11. .historyMoves() — returns array of executed moves with SAN
    12. .pgn(options?) — exports game as PGN string with optional headers
    13. .load(fen), .loadPgn(pgn) — import position/game
    14. .clear(), .reset(), .put(piece, square), .remove(square) — board manipulation

    15. .validateFen(fen) — static validation helper

    16. Create src/engine/types.ts updates:

    17. Add isCheck, isCheckmate, isCastling, isEnPassant, capturedPiece to Move
    18. Ensure all types use shorthand ('p', 'w') consistently

    Phase 2: Utility Modules (Thin Wrappers)

    1. Create src/engine/fen.ts:
    2. STARTING_FEN constant
    3. parseFEN(fen) → returns GameState with board as 8×8 array, turn, castlingRights, enPassantTarget, halfmoveClock, fullmoveNumber
    4. generateFEN(state) → returns FEN string

    5. Internally delegates to chess.js for parsing, then converts to our GameState format

    6. Create src/engine/moves.ts:

    7. generateLegalMoves(state, square) → array of legal destination squares for piece at square
    8. generateAllLegalMoves(state, color) → array of all legal moves for color
    9. isMoveLegal(state, from, to) → boolean
    10. isInCheck(state, color) → boolean
    11. isSquareAttacked(state, square, byColor) → boolean
    12. findKing(state, color) → king's square { file, rank }

    13. Internally uses a temporary Chess instance from FEN

    14. Create src/engine/special.ts:

    15. canCastleKingside(state, color), canCastleQueenside(state, color) → boolean
    16. getCastlingRookSquare(color, side) → { kingFrom, kingTo, rookFrom, rookTo }
    17. isEnPassantTarget(state, square) → boolean
    18. canPromote(from, to, color) → boolean
    19. getPromotionRank(color) → 8 (white) or 1 (black)

    20. updateCastlingRights(state, move) → updated CastlingRights

    21. Create src/engine/game.ts:

    22. createGame() → new game with starting position
    23. createGameFromFEN(fen) → game from arbitrary position
    24. makeMove(game, from, to, promotion?) → new Game with move applied (immutable)
    25. canMakeMove(game, from, to) → boolean
    26. getGameResult(state) → { result: GameResult, winner: Color \| null }
    27. isCheckmate(state, color), isStalemate(state, color), isDraw(state), isInsufficientMaterial(state) → boolean
    28. undoMove(game) → previous Game state (immutable)

    29. getMoveHistory(game) → copy of move history

    30. Create src/engine/pgn.ts:

    31. moveToSAN(move, state) → SAN string (e.g., 'Nf3+', 'O-O', 'exd5=Q')
    32. generateMoveNumber(moveNumber, color) → '1.' or '1...'
    33. generatePGN(game, headers?) → full PGN string with tags and move list
    34. Uses chess.js for SAN generation where possible

    Test Strategy

    Unit Tests (already provided in tests/tasks/T3/)

    • Board tests (board.test.ts): Coordinate conversions, square parsing, board creation, cloning, piece get/set
    • Move tests (moves.test.ts): Per-piece legal move generation, king safety (can't move into check), attack detection, findKing
    • FEN tests (fen.test.ts): Parse/generate starting position, arbitrary positions, en passant, castling rights, round-trip fidelity
    • Game tests (game.test.ts): Move execution, captures, turn switching, halfmove/fullmove clocks, castling, en passant, promotion, undo, checkmate/stalemate/insufficient material detection
    • Special tests (special.test.ts): Castling rights checks, path obstruction, king-in-check prevention, en passant validity, promotion rank detection, castling rights updates
    • PGN tests (pgn.test.ts): SAN generation for all move types, move numbering, full PGN export with headers and result
    • Integration tests (chess-engine.test.ts): Full ChessEngine class covering all public methods, edge cases, draw conditions (insufficient material, threefold repetition, 50-move rule)

    Edge Cases to Cover

    • Insufficient material: K vs K, K+B vs K, K+N vs K → draw
    • Threefold repetition: Same position reached 3 times → draw
    • 50-move rule: 50 half-moves without pawn move or capture → draw
    • Pinned pieces: Piece that would expose king cannot move
    • Discovered check: Moving a piece that blocks a check is illegal
    • Castling through check: King cannot castle if passing through attacked square
    • En passant window: Only valid immediately after double pawn push
    • Promotion default: Defaults to queen if no promotion piece specified

    Risks & Mitigations

    Risk Mitigation
    chess.js API changes in future versions Pin version in package.json (already at ^1.4.0); wrap all chess.js calls so upgrades only touch one file
    Test naming convention mismatch (shorthand vs longhand) Document the convention in src/engine/README.md; update all tests to use shorthand; add a toLonghand() helper if external APIs need it
    Board array indexing confusion ([rank][file] vs [file][rank]) Add clear JSDoc comments on board.ts functions; use named parameters (rank, file) instead of raw indices
    ChessBoard.tsx 7×7 stub breaks silently Add a data-testid="square-h8" assertion in component tests to catch grid-size regressions
    Performance of creating temporary Chess instances in utility functions Cache Chess instances where possible; benchmark with console.time; consider memoizing FEN → Chess maps
    PGN generation edge cases (ambiguous moves, disambiguation) Delegate to chess.js .pgn() for standard cases; only implement custom SAN for the moveToSAN() utility
    Server/client bundle size chess.js is ~15KB minified; tree-shake unused methods; consider lazy-loading for spectator views
