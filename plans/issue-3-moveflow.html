Move Validation & Execution Flow
MOVE LIFECYCLE
1 User drags piece from e2 to e4
2 ChessBoard.tsx calls onMove({ from: 'e2', to: 'e4' })
3 Game hook calls engine.move('e4') or canMakeMove(game, 'e2', 'e4')
4 ChessEngine delegates to chess.js for validation
✗ INVALID
Illegal move → reject
Examples: pawn backward, into check, pinned piece
✓ VALID
Execute move → update state
Generate SAN, update history
POST-MOVE CHECKS
5 Update board state, switch turn, increment clocks
6 Check for game-ending conditions:
✗ NO LEGAL MOVES
In check? → Checkmate
Not in check? → Stalemate
✗ DRAW CONDITIONS
Insufficient material
Threefold repetition
50-move rule
✓ ONGOING
Game continues
Notify opponent via WebSocket
UI UPDATE
7 Re-render ChessBoard with new position
8 Highlight last move, show check indicator if applicable
