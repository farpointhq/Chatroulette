    Issue #6: Chess Board Frontend Component

    Summary

    Replace the stubbed ChessBoard component with a full-featured interactive chess board using react-chessboard. The component must support drag-and-drop, legal move highlights, pawn promotion UI, board flipping, last-move and check indicators, and responsive sizing.

    Root Cause Analysis

    The current src/components/ChessBoard.tsx is intentionally stubbed — it renders a 7×7 grid with no pieces, no highlights, no drag/drop, and always calls onMove so every test assertion fails (RED phase). The tests in ChessBoard.test.tsx specify the full expected behavior including piece rendering, click-to-move, legal move highlighting, last move highlights, check indicators, promotion picker, board orientation, coordinate notation, and drag-and-drop support.

    The chess engine (src/engine/chess.ts) and types (src/types/chess.ts) are already in place and provide the necessary GameState, Move, and ChessEngine APIs. The component needs to consume these and render a fully interactive board.

    Proposed Solution

    Integrate react-chessboard as the rendering layer. It handles drag-and-drop, animations, square highlighting, and piece rendering out of the box. Wire it to the existing GameState prop and onMove callback. Use chess.js (already a dependency) for legal move validation via a lightweight wrapper inside the component. Implement the promotion picker as a small overlay/modal. Handle board orientation via the orientation prop. Track last move and check state from GameState.

    Files to Modify

    File Change
    package.json Add react-chessboard dependency
    src/components/ChessBoard.tsx Replace stub with full react-chessboard integration

    Implementation Steps

    1. Install dependency 

      npm install react-chessboard

    2. Update ChessBoard.tsx

    3. Import Chessboard from react-chessboard
    4. Create a Chess instance internally for move validation
    5. Map GameState.board to FEN string for the position prop
    6. Implement onPieceDrop to validate moves and call onMove
    7. Implement onSquareClick for click-to-move fallback
    8. Generate customSquareStyles for legal moves, last move, and check
    9. Render promotion picker overlay when a pawn reaches last rank
    10. Respect orientation and interactive props

    11. Apply className and responsive container styles

    12. Verify tests pass 

      npm test -- ChessBoard.test.tsx

    Test Strategy

    • Unit tests: Run ChessBoard.test.tsx which covers:
    • 8×8 board rendering (64 squares)
    • Correct piece placement from starting FEN
    • Click-to-move interaction
    • Legal move highlights on piece selection
    • Last move highlight display
    • Check indicator on king square
    • Checkmate and stalemate overlays
    • Pawn promotion picker and selection
    • Board flip on orientation="b"
    • Coordinate notation display
    • Responsive sizing
    • Empty-square click filtering
    • Drag-and-drop move making
    • interactive={false} disabling input

    • className prop support

    • Edge cases:

    • Pawn promotion with all four piece choices (Q, R, B, N)
    • Illegal move rejection (snap-back)
    • Switching orientation mid-game
    • Game over states (checkmate, stalemate)
    • Empty board / custom FEN positions

    Risks & Mitigations

    Risk Mitigation
    react-chessboard styles conflict with existing CSS Scope board within a container with isolated styles
    FEN generation from GameState mismatch Test with parseFEN helper from tests
    Promotion UI doesn't match test expectations Match data-testid attributes exactly
    Drag-and-drop events not firing in jsdom react-chessboard supports both DnD and click; tests use both
    chess.js version mismatch Already at ^1.4.0, react-chessboard is compatible
